Выношу редактируемое содержимое в один файл:
- hero (заголовок, мета-строка `6–14 лет · Анапа · ФАМ`, фон/видео path)

- goalkeeper school (текст, видео path, poster)

Это решает «контент должен быть легко редактируемым».

## 6. Форма заявки (главная UX-правка)

Заменяю текущий тяжёлый `JoinFlow` (4 шага с футбольным полем-прогрессом) на простой одностраничный `ApplicationForm`:

Поля:
- Направление/группа (select)
- ФИО ребёнка
- Дата рождения (date input)
- ФИО родителя
- Email
- Телефон
- Telegram (опционально)
- Комментарий (textarea)
- Submit «Отправить заявку»

States: default / focus / validation error / success (toast + success-state в модалке).

Открывается через **modal на desktop, drawer (bottom sheet) на mobile**, используя существующие `dialog` + `drawer` shadcn компоненты + `useIsMobile` хук.

Все CTA по сайту (`«Записать ребёнка»` / `«Оставить заявку»`) триггерят один глобальный контекст открытия формы (`ApplicationFormProvider`).

`JoinFlow.tsx` — удаляю, `ApplyModal.tsx` — заменяю на новую `ApplicationForm` + `ApplicationFormProvider`.
